# Fake News Classification — ISOT Dataset

**NLP capstone project** — can a news article be classified as fake or real using only its text?

## Overview

Misinformation spreads faster than it can be fact-checked, and manual verification doesn't scale with content volume. This project builds and compares three text classifiers — TF-IDF + Logistic Regression, TF-IDF + Linear SVM, and a fine-tuned BERT model — on the task of distinguishing fake news articles from real ones, using the ISOT Fake News Dataset.

**Research question:** How accurately can an article be classified as fake or real from text alone, and how does model choice (classical ML on TF-IDF vs. fine-tuned BERT) affect quality? Does BERT's extra computational cost pay off, and which error type (false positive vs. false negative) dominates?

## Dataset

- **Source:** [ISOT Fake News Dataset](https://onlineacademiccommunity.uvic.ca/isot/2022/11/27/fake-news-detection-datasets/), University of Victoria (Ahmed, Traore, Saad, 2018). Also mirrored on [Kaggle](https://www.kaggle.com/datasets/clmentbisaillon/fake-and-real-news-dataset).
- **Size:** 44,898 articles — 21,417 real (Reuters) / 23,481 fake (fringe/junk sites)
- **Fields:** `title`, `text`, `subject`, `date`
- **Note:** the CSV files (`True.csv`, `Fake.csv`) are not included in this repo due to size — download them from the links above and place them in the project root before running the notebook.

### Known dataset limitation — addressed directly in this project

Nearly all real articles come from a single source (Reuters) and share a distinctive style marker (a `(Reuters) —` prefix). A model can learn to recognize *source style* rather than *misinformation features*, which is a form of data leakage. This project explicitly tests for that: models are trained and evaluated both **with** and **without** the source marker, and results are compared.

## Methodology

1. **Text cleaning** — strip HTML tags, URLs, extra whitespace
2. **Two dataset versions** — original text, and a version with source-specific markers removed
3. **Split** — stratified 70% / 15% / 15% train / validation / test
4. **Models**
   - *Baseline:* TF-IDF (uni+bigrams) + Logistic Regression, and TF-IDF + Linear SVM — fast, interpretable, no context awareness
   - *Advanced:* fine-tuned `bert-base-uncased` — context-aware, more accurate, but slower and GPU-dependent
5. **Evaluation** — accuracy, precision, recall, F1 (on the fake class), and ROC-AUC, computed on the same held-out test split for every model
6. **Error analysis** — confusion matrices, qualitative review of false positive / false negative examples
7. **Interpretability** — top TF-IDF features driving the logistic regression's fake/real decisions

### Why recall (on the fake class) is the priority metric

A missed fake article (false negative) causes compounding, networked harm — it keeps spreading and shaping opinion. A wrongly-flagged real article (false positive) is a one-off, more containable harm, since legitimate sources are usually corroborated elsewhere. F1 on the fake class is the headline metric, with recall weighted as the most important component.

## Results

| Model | Accuracy | Precision | Recall | F1 | ROC-AUC |
|---|---|---|---|---|---|
| TF-IDF + LogReg (baseline) | 0.9951 | 0.9947 | 0.9950 | 0.9949 | 0.9996 |
| TF-IDF + LogReg (leakage markers removed) | 0.9892 | 0.9873 | 0.9900 | 0.9887 | 0.9989 |
| Fine-tuned BERT | 1.0000 | 1.0000 | 1.0000 | 1.0000 | 1.0000 |

*(Full metrics and plots are generated by the notebook and saved to `final_results.csv`, `model_comparison.png`, `top_features.png`, `eda_plots.png`.)*

### A caveat on the BERT result

A perfect test score is a flag worth naming rather than a result to take at face value. It's consistent with the broader literature on ISOT — several published papers report 98–99.9% but not 100% — which suggests this dataset has strong stylistic/structural artifacts beyond just the Reuters marker that make it easier than real-world fake-news detection would be. The leakage ablation above only tests one such artifact; a model this close to perfect should be read as "very good at ISOT's specific artifacts," not necessarily "solves fake news detection in general."

## Key findings

- Removing the Reuters style marker cost the logistic regression only 0.0062 F1 — the model is learning genuine textual signal, not purely source style, though leakage is a partial (not the sole) contributor to its high score.
- LogReg trades a mere 0.005 F1 for a massive cost saving (seconds on CPU vs. 20+ minutes on GPU for BERT) — for most practical deployments, the classical baseline is the pragmatic choice.
- Errors are qualitatively reviewed for both classes to check whether the model is confusing specific news topics rather than genuinely detecting misinformation cues.
